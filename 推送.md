# 项目推送命令

## 1) 日常提交并推送到 GitHub

```bash
git status
git add src/config.js src/App.vue src/app.js public/css/app.less index.html
git status
git commit -m "feat: 更新主页配置"
git push origin main
```

说明：只把本次要部署的文件写在 `git add` 后面，不要用 `git add .`。

如果你的默认分支不是 main，把最后一行改为：

```bash
git push origin master
```

## 2) 推送前先同步远程（推荐）

```bash
git pull --rebase origin main
git push origin main
```

## 3) 第一次绑定远程仓库时使用

```bash
git init
git add README.md package.json src/main.js src/App.vue
git commit -m "init: first commit"
git branch -M main
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

如果第一次提交需要全量文件，建议先确认后再执行：

```bash
git status
git add -A
git status
```

## 4) 可选：手动构建并部署到 Cloudflare Workers

项目里已有 wrangler 配置，可用下面命令手动部署静态资源：

```bash
npm install
npm run build
npx wrangler deploy
```

## 5) 常用排查命令

```bash
git remote -v
git branch
git log --oneline -n 10
```
